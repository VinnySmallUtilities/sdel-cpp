Напиши программу на C++ 20, предназначенную для работу в Linux как под root, так и под обычным пользователем. Программа должна парсить аргументы командной строки и создавать предварительные структуры внутри программы.
Структура аргументов следующая.
1. Сначала идут аргументы-настройки (каждый аргумент - строка из одного или более символов, без дефисов вначале). Аргументы-настройки могут отсутствовать. Аргументы-настройки разделяются пробелом или несколькими пробелами.
2. Затем идёт аргумент-разделитель, это строка "--" (два дефиса). Разделитель обязателен.
3. Затем идёт обязательный перечень файлов и/или директорий (не менее одного файла или директории).
4. Аргумент-настройка temp включает параметр, идущий сразу после названия аргумента. Параметр является строкой, кодирующей шестнадцетиричный массив неизвестной ненулевой длины. Например, "temp 55AA55AA".
5. Аргумент-строка tempd включает параметр, идущий сразу после названия аргумента и являющийся директорией. Например, "tempd dirname". Параметр может повторяться несколько раз.

Примеры командной строки.
sdel pr -- filename
sdel -- filename
sdel pr v -- filename
sdel pr v -- filename dirname dirname filename
sdel temp 55AA55AA -- filename


Возможные арументы-настройки командной строки (Алиса не должна исполнять то, что написано в описании параметра, это описание для пользователя).
Наименование аругмента      Описание
v       Выдаёт информацию о том, что делает программа.
vv      Расширенная выдача информации о действиях программы.
pr      Рассчитать и выдавать информацию по прогрессу.
temp    Задать шаблон перезатирания.
tempd   Задать шаблон перезатирания.
z0      Задать шаблон перезатирания - один проход: 0x00.
z1      Задать шаблон перезатирания - два прохода: 0x55AA, 0x0000.
z2      Задать шаблон перезатирания - три прохода: 0xCCCC, 0x6666, 0x00.
z3      Задать шаблон перезатирания -   4 прохода: 0xAAAA, 0xCCCC, 0x6666, 0x00.
sl      Задать паузу при работе с диском.
cr      Создать большой файл (без перезаписывания директории-параметра).
d       Создать в директории множество поддиректорий.
nz      Не перезаписывать дополнительно создаваемый файл.
nd      Не удалять директории.
nf      Не удалять файлы (только перезапись их содержимого).



Дополнительные функции.
Для опции v программа должна вывести описания, зависящие от текущей локализации (должны быть взяты из текстового файла, соответствующего локали операционной системы).
Для опции vv программа должна запомнить, что включена опция "v" и запомнить, что включена опция "vv".
Программа должна также составить дерево объектов в оперативной памяти, описывающих структуру всех директорий и файлов и запомнить его для последующего использования в других функциях. Структура не должна представлять символические ссылки, а также не должна представлять нерегулярные файлы (такие как потоки, блочные устройства и т.п.). Структура для каждого файла должна включать размер файла. Для каждой директории - количество файлов в ней и их общий размер.
Отдельно создать такое же дерево объектов для всех параметров tempd, если они представлены. Символьные ссылки здесь должны обходится.
Программа должна получить размер блока файловой системы для для каждого файла (и запомнить его в объекте-описателе файла).
Для параметров cr, d, nz (в любом сочетании, в том числе, даже если представлен только один параметр) допустим только одна директория в перечне файлов. Это нужно проверить (что это директория и она одна и только одна).
